# Cloudscape Design System Components (102)

This is a list of all 102 components available in the Cloudscape Design System, grouped by functional categories.

## Layout Components
These components provide the structure and arrangement for pages and content.
- **App layout**: Core page structure with navigation, tools, and content areas.
- **App layout toolbar**: A variant of app layout with a toolbar for productive use cases.
- **Board**: A base for configurable, drag-and-drop layouts.
- **Board components**: A set of components (Board, Board item, Items palette) for board-based interfaces.
- **Board item**: Individual draggable elements within a board.
- **Box**: A utility for typography, spacing, and basic styling.
- **Column layout**: Positions content in vertical columns.
- **Container**: A fundamental wrapper for grouping related content.
- **Content layout**: Provides structure for expressive, content-heavy pages.
- **Grid**: A responsive layout system for distributing content.
- **Items palette**: A panel for holding items to be added to a board.
- **Panel layout**: Displays two panels of content side-by-side.
- **Space between**: Manages horizontal or vertical spacing between elements.
- **Split panel**: A secondary content area that can be minimized or resized.

## Navigation Components
Components used to help users move through the application or stay oriented.
- **Anchor navigation**: Links for jumping to specific sections on a page.
- **Breadcrumb group**: Hierarchical path of the current page's location.
- **Link**: Standard interactive navigational element.
- **Pagination**: Controls for moving through multi-page content.
- **Side navigation**: Vertical menu for application-wide navigation.
- **Tabs**: Organizes content into separate views within the same context.
- **Top navigation**: Global header for consistent application-wide access.
- **Wizard**: Guides users through multi-step, complex tasks.

## Form & Input Components
Elements used for data entry, selection, and user actions.
- **Attribute editor**: For creating, editing, and deleting key-value attributes.
- **Autosuggest**: Input with a list of matching suggestions.
- **Button**: Primary action trigger.
- **Button dropdown**: Groups multiple actions under a single button.
- **Button group**: A set of related buttons displayed together.
- **Calendar**: For selecting dates from a grid.
- **Checkbox**: Allows selecting one or more options.
- **Code editor**: Specialized input for writing and editing code.
- **Code view**: Read-only display of formatted code.
- **Collection preferences**: Configures display settings for tables or cards.
- **Collection select filter**: Filter for selecting specific items from a list.
- **Date input**: Text input specifically for dates.
- **Date picker**: Popover-based date selection.
- **Date range picker**: For selecting a span between two dates.
- **File dropzone**: Area for dragging and dropping files.
- **File input**: Trigger for selecting files from a local device.
- **File token group**: Displays selected files as removable tokens.
- **File upload**: Comprehensive component for handling file selection and uploads.
- **Form**: Wrapper for grouping and submitting input fields.
- **Form field**: Provides labels, descriptions, and error messages for inputs.
- **Input**: Standard single-line text entry.
- **Multiselect**: Allows selecting multiple items from a list.
- **Prompt input**: Specialized input for Generative AI prompts.
- **Property filter**: Complex filtering for data collections.
- **Radio group**: Allows selecting exactly one option from a set.
- **Select**: Dropdown for choosing a single option.
- **Slider**: For selecting a value from a continuous range.
- **Text area**: Multi-line text entry.
- **Text filter**: Simple text-based search for collections.
- **Tiles**: Selection using large, visual cards.
- **Time input**: Input for absolute time values.
- **Toggle**: Immediate on/off switch for settings.
- **Toggle button**: Buttons that maintain an "active" or "pressed" state.
- **Token group**: Displays a set of removable data points (tokens).

## Content & Data Display Components
Components for displaying data, status, and informational content.
- **Avatar**: Visual representation of a user or entity.
- **Badge**: Small color-coded label for categorization or severity.
- **Cards**: Displays collection items in a visual, grid-friendly format.
- **Chat bubble**: Represents a single message in a conversational interface.
- **Copy to clipboard**: Trigger for copying text to the user's clipboard.
- **Header**: Defines titles and actions for sections or containers.
- **Key-value pairs**: Displays data in a label-value format.
- **List**: Simple vertical arrangement of items.
- **S3 resource selector**: Navigates and selects files from Amazon S3.
- **Table**: Comprehensive component for displaying and managing tabular data.
- **Tag editor**: For managing a list of tags.
- **Text content**: Applies typography styles to standard HTML elements.
- **Token**: Individual compact representation of a data point.
- **Tree view**: Hierarchical list of nested items.

## Feedback & Status Components
Used to communicate system state and provide contextual information.
- **Alert**: Inline message for instructions or information.
- **Drawer**: Slidable content area for supplementary information.
- **Error boundary**: Gracefully handles and displays UI crashes.
- **Expandable section**: Collapsible container for hiding or showing content.
- **Flashbar**: Stacked notifications for system-wide status.
- **Help panel**: Contextual documentation and help information.
- **Hotspot**: Anchor for contextual help or tutorials.
- **Icon**: Visual symbols used for metaphors and actions.
- **Live region**: Communicates dynamic updates to assistive technologies.
- **Loading bar**: Indicates a background process is active.
- **Modal**: Focused dialog that requires user interaction.
- **Popover**: Floating content that appears on hover or click.
- **Progress bar**: Visual indicator of completion for long-running tasks.
- **Spinner**: Indicates content or a page is currently loading.
- **Status indicator**: Communicates specific states (Success, Error, etc.).
- **Steps**: Progress tracker for multi-stage processes.
- **Support prompt group**: Help and feedback triggers for specific workflows.

## Data Visualization (Charts) Components
Specialized components for representing data graphically.
- **Area chart**: Visualizes stacked data series over time.
- **Bar chart**: Visualizes data series with emphasis on totals.
- **Cartesian charts**: Base component for X-Y coordinate charts.
- **Charts**: Top-level entry for chart collections.
- **Charts (Legacy)**: Older versions of chart components.
- **Line chart**: Emphasizes trends in data over time.
- **Mixed line and bar chart**: Combines both bar and line visualizations.
- **Pie and donut charts**: Visualizes part-to-whole relationships.
- **Pie and donut charts (Legacy)**: Older versions of pie/donut charts.

## Generative AI & Tutorial Components
- **Generative AI components**: Specialized set including Prompt input and Chat bubble.
- **Tutorial components**: Group of components for hands-on onboarding.
- **Tutorial panel**: Interface housing contextual onboarding instructions.
